# Avito Help Search

Для каждого запроса из `test.f` ранжирую статьи справки и возвращаю
10 `article_id`. Целевая метрика — `MAP@10`.

## Результаты

| Подход | OOF MAP@10 | TestSys |
|---|---:|---:|
| Popularity | 0.3159 | 0.27 |
| Stemmed BM25 | 0.3187 | — |
| Query kNN | 0.6008 | — |
| E5 + char kNN | 0.6035 | — |
| LightGBM LambdaMART | 0.6910 | — |
| **E5 + LightGBM blend** | **0.6928** | **0.603060** |

## Подход

1. Из HTML извлекаю видимый текст, заголовки, ссылки, `alt` и начало статьи.
   Отдельно строю stemmed-представление.
2. Получаю признаки из BM25, word/char TF-IDF, query kNN, линейных моделей,
   профилей запросов и связей между статьями.
3. Для пар `запрос — статья` считаю score-, rank-, overlap- и length-признаки.
   Два LightGBM LambdaMART обучаю с разными seed.
4. Локальная `multilingual-e5-small` кодирует запросы. Метки переношу от
   20 ближайших calibration-запросов, смешивая E5 cosine и char TF-IDF как
   `0.85 / 0.15`. Степень 8 усиливает близких соседей, деление на частоту
   метки в степени `0.3` ослабляет перекос к самым частым статьям.
5. Финальный score — `0.8` среднего reciprocal rank двух LightGBM и `0.2`
   reciprocal rank семантического kNN. Равные веса LightGBM уменьшают
   зависимость от seed; небольшой вес E5 добавляет новый сигнал и не ломает
   сильный LTR-порядок. Эта простая комбинация улучшила nested OOF на всех
   folds, кроме одного.

## Валидация и ошибки

Использую 5-fold nested OOF с `shuffle=True` и `random_state=42`.
Supervised query-признаки для каждого fold строятся только по train-части.
Веса финального бленда выбраны по OOF, поэтому `0.6928` не является оценкой
на отдельном holdout.

Основные ошибки возникают на редких статьях, коротких неоднозначных запросах
и вопросах с несколькими намерениями. Char TF-IDF помогает с опечатками,
E5 — с разговорными перефразировками, а LambdaMART объединяет эти сигналы.

Использую только выданные Feather-файлы и локальную open-source модель
`intfloat/multilingual-e5-small` на 117.7 млн параметров. Внешние API,
ручная разметка `test.f` и ответы для отдельных `query_id` не используются.

## Запуск

Все команды и ноутбуки запускаются из корня репозитория. Проверенное
окружение — Python `3.14.6`; версии библиотек зафиксированы.

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
python run.py
```

При первом запуске скачиваются зафиксированные веса E5; дальше модель работает
локально. `run.py` проверяет формат и сохраняет `outputs/answer.csv`.
В ноутбуках `00–08` сохранение submission закомментировано, в
`09_final_pipeline.ipynb` — включено.
